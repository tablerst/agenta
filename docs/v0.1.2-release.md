# v0.1.2 发布与打包说明

`v0.1.2` 是 SearchQuery RAG 化、Agent-first 工作流收敛、运行时错误日志和数据库迁移诊断的收口版本。`v0.1.1` 的 SearchV2 发布说明仍保留在 [search-v2-release.md](search-v2-release.md)，本说明只记录当前版本的打包口径。

## 发布范围

已纳入 `v0.1.2`：

- SearchQuery RAG 化：保留 task 粗召回，补齐 activity chunk 级向量、二跳 evidence、semantic telemetry、轻量 context 恢复和 evidence-aware rerank 基线。
- SearchQuery 评测：新增临时 SQLite + mock Chroma/embedding 的程序化评测套件，输出 accuracy、recall、MRR、证据覆盖和耗时指标。
- Agent-first 工作流：项目长期上下文继续由仓库文件承载，Agenta 作为任务级 ledger、closeout 和 evidence surface；默认 MCP 工具面保持 task-critical。
- 应用级错误日志：新增 `paths.error_log`，Desktop release 启动失败、Tauri command、CLI、standalone MCP 和 panic hook 统一写入 JSONL。
- 数据库迁移诊断：SQLx migration 失败现在映射为 `migration_error`，details 包含 `kind`、`version`、`retryable=false` 和备份/恢复指引；SQLite busy 仍保持 `storage_busy`。

不纳入当前版本：

- 自动修复 `_sqlx_migrations`。
- 自动同步/复制本地向量库或派生 chunk。
- 新增默认 MCP sync 运维工具。
- 真实 embedding provider 的线上质量门槛；当前评测仍以确定性 mock 管线为主。

## 打包闸口

打包前至少执行：

```powershell
bun run build
cargo check --manifest-path src-tauri/Cargo.toml
cargo test --manifest-path src-tauri/Cargo.toml
```

建议额外执行：

```powershell
cargo test --manifest-path src-tauri/Cargo.toml search_eval_v011_reports_relevance_and_performance_metrics -- --nocapture
cargo test --manifest-path src-tauri/Cargo.toml migration_checksum_mismatch_returns_structured_error
bun run release -- --dry-run
```

通过标准：

- 前端类型检查和 Vite build 通过。
- Rust check/test 无警告、无失败。
- Search eval mock 管线保持现有 relevance 与 evidence coverage 门槛。
- migration checksum mismatch 能返回 `migration_error`，且不会被误归为普通 `internal_error`。
- `bun run release -- --dry-run` 展示版本为 `0.1.2`，目标 artifact 目录符合预期。

## 迁移与回滚

- 已发布或已被用户库执行过的 migration 文件不能修改；schema 变更必须新增更高编号 migration。
- 遇到 `migration_error` 时，先备份数据库，再根据 `details.kind` 和 `details.version` 判断是 checksum mismatch、dirty migration、缺失版本还是执行失败。
- 不要直接删除用户数据库，也不要直接编辑 `_sqlx_migrations`，除非已有经过验证的修复计划。
- Search/RAG 的向量库和 chunk 仍是本地派生状态；回滚时优先关闭 `search.vector.enabled`，不要把向量库当成同步权威数据。
