After code changes, run the smallest relevant verification set: `npm run build` or `bun run build` for frontend/type-check coverage, `cargo check --manifest-path src-tauri/Cargo.toml` for Rust updates, and `cargo test --manifest-path src-tauri/Cargo.toml` when Rust tests are present. For UI or permission changes, also verify behavior with `npm run tauri dev` or `bun run tauri dev`. Avoid editing generated outputs such as `dist/`, `target/`, `src-tauri/target/`, and `src-tauri/gen/` unless regeneration is the task.
