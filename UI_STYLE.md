# Agenta - 桌面端视觉指引 (UI_STYLE.md)

## 1. 视觉主题 (Visual Identity)

Agenta 是一款以 CLI 和 AI 协作为核心的现代化工具。其视觉语言深度借鉴 **Linear 的黑白极简美学**，呈现出“克制、锐利、专业、高信息密度”的质感。

### 1.1 主题切换与光感基调 (Theme & Illumination)
- **双主题无缝支持 (Light / Dark Mode)**：原生支持系统级的明暗模式自动切换。无论是在深邃的暗色环境中，还是在通透的亮色背景下，都能保持高密度、极简洗练的 Linear 质感与一致的高阶光影视觉（Lighting Effects）。
- **色彩与光影策略 (Light over Color)**：**极致去色，以光代色**。界面的层级与交互反馈不再过度依赖传统的“纯色背景变换”，而是引入微观真实的“光感与质感”。依靠极弱的发光边框 (Highlighted Borders)、渐变流光 (Glows) 和指针聚光灯跟随 (Spotlight Tracking) 来塑造高级交互反馈。

## 2. 视觉 Token 规范 (Visual Tokens)

### 2.1 颜色、材质与光影 (Colors & Lighting)
采用 Linear 风格的极致高对比黑白调色板，并融入微妙的光源映射模拟细腻质感。

*   **背景色与层级光感 (Backgrounds)**
    *   主视图基础底色 (Main Background): `Light: #FFFFFF` / `Dark: #0A0A0A` (极净的高级深黑/纯白幕布底色)
    *   侧边栏与悬浮层 (Elevated Surfaces): 利用含有微小冷暖倾向的极简灰阶拉开空间感。`Light: #F7F7F8` (带有微微暖意的极浅灰白) / `Dark: #141415` (深邃但透着一丝呼吸感的炭灰，顶部附加内高光 `box-shadow: inset 0 1px 0 rgba(255,255,255,0.03)` 以凸显立体感)。
    *   光感线与边框 (Glowing Borders): `Light: rgba(0,0,0,0.04)` / `Dark: rgba(255,255,255,0.06)`。摒弃粗重的阴影，重点浮层或组件只需保留顶侧或者边缘的 `1px` 高光截面线（流光线），去模拟物理世界的反光边缘。
    *   交互悬浮 (Interactive Hover & Spotlight): **鼠标光斑跟随**。鼠标移到列表项、按钮组合区时，使用动态跟随的极弱径向渐变（Radial Gradient）在鼠标下方产生一层薄光晕聚焦（亮色中为冷灰光，暗色中为冷白微光），而不是死气沉沉的单色块贴片。
*   **文本色 (Text & Content)**
    *   主要大字与标题 (High Contrast): `Light: #0F0F0F` / `Dark: #EDEDED` (纯净锐利)
    *   次要文本与描述 (Medium Contrast): `Light: #737373` / `Dark: #A1A1AA` (清晰细腻，拉开灰阶层次)
*   **强调与状态色 (Accent & States - 极度克制)**
    *   主品牌色/聚焦 (Primary Accent): **黑白反转** 或 低饱和度的浅色调。避免大面积蓝紫色块覆盖，保持界面冷峻感。
    *   阻断与警告 (Policy Deny / Errors): `#EF4444` -> (视觉降噪后) `#F87171`，仅点缀于 Icon 边缘或细边框，配合极低透明度的背景，保持视觉通透不粘腻。
    *   审批与动作 (Require Human): 低饱和度的琥珀色/黄色。带有微弱呼吸感的内发光提示。
    *   静默/通过 (Auto / Allow): 低饱和度的柔和绿色。
*   **代码高亮配色 (Syntax Highlighting - 专属克制版)**
    *   废除 VS Code 默认强烈的蓝/绿原色配比。对 JSON/Payload 以及日志块，实行低饱和度的灰度衍生色彩：
    *   `Dark`: 采用幽静紫蓝处理 Key，蒙尘薄绿处理 String，配合灰桔色数字与灰色标点。
    *   `Light`: 采取相对应的克制色彩降噪搭配，以提供像印在纸质书框中的艺术代码块质感。

### 2.2 排版与字体 (Typography)
因为涉及到大量的文本摘要 (`task_search_summary`)、日志记录和代码，字体选择至关重要。

*   **界面主字体 (UI Font)**: 首选系统原生的无衬线字体以保证最佳清晰度和性能 (`Inter`, `San Francisco`, `Segoe UI`, `Roboto`)。
*   **代码与终​​端字体 (Monospace Font)**: 凡涉及 MCP payload、附件纯文本、摘要字段内容，强制使用等宽字体 (`JetBrains Mono`, `Fira Code`, `SF Mono`)。
*   **字阶规则**：
    *   H1 / H2 控制在 `20-24px` 级别，字重 `600`。
    *   正文控制在 `14px`，保持 `1.5` 的理想行高。
    *   元信息（时间戳、版本号）为 `12px`，字重 `400`。

### 2.3 间距与圆角 (Spacing & Radius)
*   **圆角 (Border Radius)**：以精致、洗练的几何感为主，体现硬朗。面板、输入框等容器默认采用 `4px` ~ `6px` 小圆角，不用大圆角（拒绝可爱的果冻感）。
*   **空间网格 (Grid & Spacing)**：基于标准的 `4px / 8px` 网格系统，增加整体面板留白（Padding），以低密度的边距容纳高密度的文字（典型的 Linear 排版技巧）。

## 3. 组件视觉基准 (Component Baselines)

### 3.1 按钮与控件 (Controls & Glare)
- **边框流光点按反馈**：绝大多数常规级行动点使用无底色 (`var(--bg-sidebar)`) 配合高光描边的高级样式。包含极其微弱的外围阴影 (`0 1px 2px`) 带来立体感；按压 (Active) 时迅速叠加逼真的深凹陷感。
- **Primary打字机琴键效果**：取消原生浓烈的彩色，`Primary Action`（新建项目、批准指令）需要通过“全黑白反转”的色块，加之顶部白光反射切线 (`inset 0 1px 0 rgba(255...`) + 极重的底侧投影。这使得该主干道按钮犹如真实的琴键一样有着机械爽快感。

### 3.2 徽章与卡片层 (Badges & Glass Surfaces)
- **写策略告警卡 (Policy Cards)**: 彻底摒弃粗暴死板的色块背景或者刺眼的红黄外框。改为带 `180deg` 轻微上下透射光的上下渐变透明处理，结合 `inset` 的玻璃边缘内泛白光和低浊底色，在不喧闹中暗示最高的高阶警惕与信息密集度。
- 策略徽章呈现：
    - `auto`: 纯幽灵灰。
    - `require_human`: 低照度琥珀色边内光缓慢呼吸。
    - `deny`: 黯红滤镜加高级悬浮微光气泡。

## 4. 视觉反模式 (Anti-patterns)
- **拒绝塑料感的扁平死色**：纯粹的大色块容易显得廉价且无法区分亮深主题的层级。即便是基础的灰白黑组件，也必须配以光照层次（边框高光、微妙内泛光、跟随指引光斑）才能在暗色与亮色模式下展现高级感与立体感。
- **拒绝低俗的炫彩跑马灯与粗暴的大阴影**：严禁诸如各种高饱和度红蓝霓虹变色渐变。由于我们的主题追求极简极客属性，所有光感都必须是“干净通透、在 `0.1` 以下的不透明度的纯净冷白光或深素灰压屏暗光”。
- **无多余阻滞物理惯性**：鼠标或键盘位移触发的光斑必须瞬发顺滑，零果冻、零弹簧回弹。任何状态过渡或发光变暗必须锁死在干净清晰的 `100ms-150ms` 区间内流转。
