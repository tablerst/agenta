# 桌面端项目工作区与滚动语义重构

## 背景

当前桌面端原本采用 `Projects / Tasks / Approvals / Runtime` 并列一级导航，带来三个直接问题：

- 项目级上下文被全局页面切碎，任务与审批经常丢失项目归属。
- `Projects` 页面同时塞入项目编辑、版本列表、创建版本和版本编辑，导致信息层级混乱。
- 桌面端缺少统一的 `100dvh + min-height: 0 + overflow: hidden` 约束，长表单、长 JSON、长审批载荷会把整页顶出视口。

本轮目标是把桌面端收敛为“全局壳层 + 项目工作区”模型，并同时修正导航语义、滚动语义和审批项目归属契约。

## 方案

### 交互结构

- 全局侧边栏只保留 `Projects`、`Runtime`、全局搜索、语言、主题。
- 项目工作区采用父壳层：左侧项目注册表，右侧项目头部 + 项目级二级导航 + 工作区内容。
- 项目级二级区段固定为 `概览 / 版本 / 任务 / 审批`，通过路由切换。
- `Tasks` 和 `Approvals` 保留旧入口作为兼容重定向，不再在侧边栏显示。

### 滚动与布局

- 桌面端根壳层锁定为 `100dvh`。
- 侧边栏、项目注册表、工作区列表列、详情检查器列全部独立滚动。
- 长 JSON、长 payload 和长列表必须局限在所属面板内滚动，不再撑高整页。
- 移动端保留侧边栏抽屉化，但不再维持桌面端多列并发滚动压缩布局。

### 语义与契约

- 任务详情内部的 `概览 / 备注 / 附件 / 活动` 改为真实 `tablist / tab / tabpanel`。
- 审批状态与范围切换使用 filter chips，不复用 tab 语义。
- 审批查询契约新增项目过滤，审批响应补齐 `project_ref / project_name / task_ref`。
- 审批所属项目与资源回跳目标统一由服务层解析，前端不再猜测。

## 执行步骤

### Phase 1：壳层与路由重组

- 调整侧边栏，仅保留全局能力入口。
- 引入项目工作区父壳层与子路由：
  - `/projects/:projectSlug/overview`
  - `/projects/:projectSlug/versions`
  - `/projects/:projectSlug/tasks`
  - `/projects/:projectSlug/approvals`
- 保留 `/tasks`、`/approvals` 为兼容重定向入口。

### Phase 2：工作区分栏与语义修正

- 为项目工作区新增 sticky 头部、项目级二级导航、列表列和检查器列。
- 把版本、任务、审批拆到项目工作区内部。
- 将任务详情按钮组替换为真实 tab 语义。
- 将审批状态/范围切换替换为 chips。

### Phase 3：审批项目感知与回跳

- 扩展 `ApprovalQuery` 与 `ApprovalRequest` 契约。
- 在服务层统一解析审批所属项目、项目名和任务引用。
- 统一创建后、审批入队后、审批回跳后的项目内导航行为。

### Phase 4：验证与收尾

- 运行前端构建与 Rust 检查。
- 运行 Rust 测试覆盖审批项目过滤与字段回填。
- 补充文档与后续手动验收记录。
